# 🌀 Entre Planos
Entre Planos é um guia de regras para Aventuras Compartilhadas, desenvolvido a partir da base estrutural do Dungeons & Dragons (Quinta Edição, 2024). O sistema propõe um conjunto de diretrizes para worldbuilding colaborativo e gestão de guildas, permitindo que múltiplas mesas coexistam dentro de um mesmo universo narrativo compartilhado.

Inspirado nos suplementos Planeshift, de James Wyatt, este guia oferece ferramentas para adaptação de cenários e integração de diferentes planos de existência. Assim como na Adventurers League, estabelece normas comuns para progressão de personagens, uso de recursos e impacto coletivo das ações dos jogadores, garantindo consistência e equilíbrio entre aventuras conduzidas por diferentes Mestres.

O objetivo central do Entre Planos é criar um ecossistema de jogo no qual cada campanha individual contribui para o desenvolvimento de um mundo persistente e interconectado, em que as escolhas feitas por um grupo de aventureiros podem reverberar em todo o cenário.

## 📦 Como baixar o projeto
Siga os passos abaixo para baixar o projeto no seu computador:

### 1. Instalar o Git
Se você ainda não tem o Git instalado:

* Baixe em: [https://git-scm.com/](https://git-scm.com/)
* Instale normalmente (pode clicar em "Next" em tudo)

### 2. Clonar o repositório
Abra o terminal (Prompt de Comando, PowerShell ou Git Bash) e execute:

```bash
git clone https://github.com/yobwarzy/entre_planos_repository.git
```

Isso vai criar uma pasta com o projeto no seu computador.
### 3. Entrar na pasta do projeto

```bash
cd entre_planos_repository
```

---

## 🔄 Como atualizar o projeto
Sempre que houver atualizações no repositório, você pode baixar as mudanças com o comando abaixo:

```bash
git pull
```

Isso vai atualizar os arquivos do seu projeto com a versão mais recente.

---

## 🏷️ Como atualizar a versão do projeto
Se você quiser atualizar a versão do projeto (ex: v1.0 → v1.1), siga este passo a passo simples:

### 1. Fazer alterações no projeto
Edite os arquivos normalmente.
### 2. Adicionar os arquivos modificados

```bash
git add .
```

### 3. Criar um commit (salvar versão)

```bash
git commit -m "Atualizando projeto para nova versão"
```

### 4. Enviar para o GitHub

```bash
git push
```

---

## 🏷️ (Opcional) Criar uma versão numerada (Tag)
Se quiser marcar uma versão oficial:

```bash
git tag v1.0
git push origin v1.0
```

---

## ⚠️ Dicas importantes
* Sempre rode `git pull` antes de começar a trabalhar
* Evite editar os mesmos arquivos em dois lugares diferentes ao mesmo tempo
* Faça commits com mensagens claras
